# 限制并发请求数

### 实现什么效果？

直接`promise.all`，则所有请求会并发处理。



### 有什么思路？

- `promise.all`

  `promise.all`并发数n个请求，返回之后，继续`promise.all`。

  问题是效率低，不能时刻按最大并发数执行。

- 一次开启并发数n个请求，完成一个，新开一个
